## Purpose
Create a dashboard view that displays pipeline execution metrics and run summaries.

## Responsibilities
- Display pipeline run summaries
- Show record counts and processing statistics
- Visualize validation results (valid vs invalid record breakdown)

## Expected files/modules
- `modules/ui/views/metrics.py` (or equivalent)
- Integration with pipeline run summary output (PS-043)
- Integration with `modules/persistence/persistence_manager.py`

## Acceptance criteria
- Dashboard displays pipeline metrics including record counts
- Metrics update when the pipeline is run
- Validation result breakdown is visible

## Short implementation notes
Read from the persisted run summary or API pipeline status endpoint. Display key metrics using charts or summary cards: records ingested, valid, invalid, and persisted. Refresh on page reload or on a manual trigger.
