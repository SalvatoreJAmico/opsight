## Purpose
Expose the operational state and last run summary of the pipeline through an API endpoint.

## Responsibilities
- Report pipeline health
- Return last run summary
- Provide basic execution metrics

## Expected files/modules
- `modules/api/routes/status.py` (or equivalent)
- Integration with pipeline run summary output (PS-043)

## Example endpoint
```
GET /pipeline/status
```

## Acceptance criteria
- Endpoint returns pipeline status
- Response contains summary metrics (records ingested, valid, invalid, persisted)
- Returns a structured JSON response

## Short implementation notes
Store or cache the most recent pipeline run summary and return it from this endpoint. If no run has been executed yet, return a clear `{"status": "no runs recorded"}` response. Summary structure should match the PS-043 output format.
