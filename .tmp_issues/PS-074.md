## Purpose
Provide evaluation metrics for the anomaly detection models to support tuning, monitoring, and reporting.

## Responsibilities
- Measure anomaly frequency across pipeline runs
- Track model performance indicators
- Provide evaluation summaries consumable by the dashboard or API

## Expected files/modules
- `modules/intelligence/evaluation.py`
- Consumes outputs from `modules/intelligence/anomaly_detection.py` (PS-072)
- Integration with `modules/intelligence/scoring.py` (PS-073)
- Outputs available to visualization layer (PS-064)

## Acceptance criteria
- Evaluation metrics are generated after anomaly detection runs
- Metrics are available for visualization or reporting
- Summary includes at minimum: anomaly rate, score distribution, and severity breakdown

## Short implementation notes
Implement an `evaluate(scored_records)` function that returns a metrics dict including `total_records`, `anomaly_count`, `anomaly_rate`, `severity_breakdown`, and `score_distribution`. Keep this function pure (no side effects) so it can be called from both the pipeline runner and the API layer.
