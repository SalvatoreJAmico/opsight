## Purpose
Implement anomaly detection logic for identifying unusual records or system behavior in Opsight pipeline data.

## Responsibilities
- Compute anomaly scores for processed records
- Identify and return outlier records
- Support configurable detection strategies

## Possible approaches
- Statistical thresholds (z-score, IQR)
- Isolation Forest
- Clustering-based anomaly detection (e.g. DBSCAN)

## Expected files/modules
- `modules/intelligence/anomaly_detection.py`
- Consumes feature sets from `modules/intelligence/feature_engineering.py` (PS-071)
- Outputs anomaly scores consumable by `modules/intelligence/scoring.py` (PS-073)

## Acceptance criteria
- Anomaly scores are produced for a set of input records
- Outlier records can be detected and returned with their scores
- Detection strategy is configurable (at minimum supports one statistical and one model-based approach)

## Short implementation notes
Implement a `detect_anomalies(features)` function that returns a list of records with anomaly scores attached. Start with a statistical baseline (z-score) and add Isolation Forest as a configurable alternative. Scores should be normalized to a 0–1 range for downstream use in the scoring system.

Academic alignment note: This module may serve as the core deliverable for the machine learning course project, including unsupervised anomaly detection and model-driven analysis of operational data.
