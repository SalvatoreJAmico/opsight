## Purpose
Prepare the runtime environment required to deploy and operate the Opsight system outside of development.

## Responsibilities
- Define production configuration separate from development settings
- Configure environment variables for all services
- Prepare Azure resource integration (storage, compute, secrets)
- Ensure all services can run using production configuration

## Expected files/modules
- `configs/production.yaml` or `configs/production.env`
- `modules/config/storage_config.py` — review and extend for production targets
- `.env.example` — document all required environment variables
- Integration with Azure services where applicable

## Acceptance criteria
- Production configuration exists and is documented
- Environment setup is documented so it can be reproduced
- System can start and run using production configuration without dev-only settings

## Short implementation notes
Separate config concerns by environment (dev vs production). Use environment variables for all secrets and connection strings — never hardcode them. Provide a `.env.example` file that documents every variable the system needs without including real credentials.
