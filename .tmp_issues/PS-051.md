## Purpose
Expose an API endpoint that allows external systems to submit data to the pipeline.

## Responsibilities
- Accept structured JSON input data
- Trigger ingestion and pipeline processing
- Validate input structure before processing

## Expected files/modules
- `modules/api/routes/ingest.py` (or equivalent)
- Integration with `modules/ingestion/ingestion.py`
- Integration with pipeline orchestrator

## Example endpoint
```
POST /data
```

## Acceptance criteria
- Endpoint accepts JSON input
- Data is passed to the ingestion stage
- Validation occurs before persistence
- Returns a structured response confirming ingestion outcome

## Short implementation notes
Deserialize the incoming JSON request body and pass it to the ingestion module. Trigger the full pipeline and return a summary response. Return HTTP 422 for malformed input and HTTP 500 for pipeline failures.
