## Purpose
Prepare the Opsight system to run in containers so it can be deployed consistently across environments.

## Responsibilities
- Create a Dockerfile for the pipeline and API services
- Define all runtime dependencies
- Support containerized execution of the pipeline runner and API service

## Expected files/modules
- `Dockerfile` — container image definition
- `docker-compose.yml` — optional multi-service orchestration
- `requirements.txt` — must be complete and pinned for reproducible builds

## Acceptance criteria
- Container builds successfully from the Dockerfile
- Container can run the pipeline (`python run_pipeline.py`) and/or start the API
- No dev-only dependencies are required inside the container

## Short implementation notes
Use a slim Python base image (`python:3.11-slim` or equivalent). Copy only necessary source files and install from `requirements.txt`. Define a default `CMD` that starts the API service. Ensure the container can be rebuilt cleanly without cached layers for CI purposes.
