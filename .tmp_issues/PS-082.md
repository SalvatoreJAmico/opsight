## Purpose
Automate testing and build processes for the repository using GitHub Actions so quality checks run on every commit.

## Responsibilities
- Configure a GitHub Actions workflow for the repository
- Run the full test suite automatically on push and pull request
- Enforce passing tests before merges are allowed

## Expected files/modules
- `.github/workflows/ci.yml` — CI workflow definition
- Integration with `pytest` (PS-081)
- Optional: linting step (e.g. `flake8` or `ruff`)

## Acceptance criteria
- CI pipeline triggers automatically on push to main and on pull requests
- Tests are executed automatically as part of the workflow
- Workflow configuration is stored in the repository

## Short implementation notes
Define a workflow with a single `test` job that checks out the repo, installs dependencies from `requirements.txt`, and runs `pytest`. Keep the workflow minimal at first — testing passes cleanly is the priority before adding linting or coverage gates.
