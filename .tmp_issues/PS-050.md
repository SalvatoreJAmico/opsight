## Purpose
Create the API service layer for Opsight using a lightweight web framework.

## Responsibilities
- Initialize API service
- Configure routing
- Integrate with pipeline modules
- Support service startup

## Expected files/modules
- `modules/api/` (new directory)
- `modules/api/app.py` — API application entry point
- `modules/api/routes.py` — route definitions
- Integration with `modules/pipeline/` or `run_pipeline.py`

Preferred tool: FastAPI (or Flask as fallback)

## Acceptance criteria
- API server starts successfully
- Service can run locally
- Base endpoint returns a health/status response

## Short implementation notes
Bootstrap a FastAPI application with a `/health` or `/status` root endpoint that returns `{"status": "ok"}`. Keep routing modular so each feature endpoint (ingestion, retrieval, status) can be defined in its own file and registered via a router.
