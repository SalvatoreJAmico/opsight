## Purpose
Create the visualization service that provides a human-friendly interface for interacting with Opsight data.

## Responsibilities
- Initialize visualization environment
- Configure dashboard framework
- Connect visualization layer to the API or persistence layer

## Expected files/modules
- `modules/ui/` (new directory)
- `modules/ui/app.py` — dashboard application entry point
- Integration with `modules/persistence/persistence_manager.py` or the Phase 5 API layer

Suggested tool: Streamlit (or lightweight web dashboard framework)

## Acceptance criteria
- Visualization service starts successfully
- Dashboard can connect to the Opsight data source
- A basic page loads with placeholder content

## Short implementation notes
Bootstrap a Streamlit app (or equivalent) with a landing page that confirms connectivity to the data source. Structure the app so each dashboard view (metrics, entity explorer, validation errors, anomalies) lives in its own module and is imported into the main app file.
