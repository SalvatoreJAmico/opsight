## Purpose
Create visual tools for inspecting validation failures so users can identify and act on data quality issues.

## Responsibilities
- Display validation error counts per run
- Categorize errors by type (e.g. missing fields, type mismatches, duplicates, timestamp errors)
- Show example records that triggered each error type

## Expected files/modules
- `modules/ui/views/validation_errors.py` (or equivalent)
- Integration with `modules/validation/quality_report.py`
- Integration with `modules/validation/report.py`

## Acceptance criteria
- Validation errors are visible in the dashboard
- Errors are categorized by type with counts
- Users can identify which records are problematic

## Short implementation notes
Read from validation quality reports generated during pipeline runs. Render error breakdowns as bar charts or summary tables. Allow drill-down to see sample records for each error category.
