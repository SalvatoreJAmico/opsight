## Purpose
Implement consistent, centralized error handling across the API service layer.

## Responsibilities
- Return structured error responses for all failure modes
- Prevent server crashes from unhandled exceptions
- Log API-level errors clearly with context

## Expected files/modules
- `modules/api/errors.py` (or equivalent error handler module)
- Applied globally across all API routes

## Acceptance criteria
- API returns clear, structured error messages for all failure cases
- Invalid or malformed requests are handled safely and return informative responses
- Unhandled exceptions do not cause bare 500 responses

## Short implementation notes
Register a global exception handler in FastAPI (or equivalent in Flask) that catches unhandled exceptions and returns a consistent JSON error envelope: `{"error": "<message>", "detail": "<context>"}`. Use HTTP 400 for input errors, 404 for not found, and 500 for internal failures. Log all errors with stage context before returning.
