## Purpose
Provide an API endpoint to retrieve stored entity records from the persistence layer.

## Responsibilities
- Query stored canonical records by entity ID
- Return entity history or data points
- Handle missing or unknown entity IDs gracefully

## Expected files/modules
- `modules/api/routes/entities.py` (or equivalent)
- Integration with `modules/persistence/persistence_manager.py`

## Example endpoint
```
GET /entity/{entity_id}
```

## Acceptance criteria
- Endpoint returns stored records for a valid entity ID
- Response is JSON formatted
- Returns a 404 response for unknown entity IDs

## Short implementation notes
Use the persistence manager to look up records by entity ID. Return the canonical record list as a JSON response. If no records are found, return HTTP 404 with a clear message rather than an empty 200.
