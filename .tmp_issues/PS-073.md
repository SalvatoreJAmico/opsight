## Purpose
Create a scoring system that converts raw anomaly results into actionable severity levels and alerts.

## Responsibilities
- Assign anomaly scores to records
- Categorize records into severity levels (e.g. low, medium, high, critical)
- Generate alert flags for records that exceed thresholds

## Expected files/modules
- `modules/intelligence/scoring.py`
- Consumes anomaly scores from `modules/intelligence/anomaly_detection.py` (PS-072)
- Outputs are surfaced through API (PS-053 / PS-05x) or dashboard (PS-064)

## Acceptance criteria
- Anomaly scoring logic is implemented and produces severity categories
- Alert categories are defined and assigned consistently
- Alert flags are visible through the API or dashboard layer

## Short implementation notes
Implement a `score_records(anomaly_results)` function that maps raw anomaly scores to severity buckets using configurable thresholds. Return a list of scored records with `anomaly_score`, `severity`, and `alert` fields attached. Thresholds should be defined in a config or as module constants for easy tuning.
