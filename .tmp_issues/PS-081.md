## Purpose
Implement an automated testing framework to ensure system reliability across all pipeline stages and API endpoints.

## Responsibilities
- Add integration tests for each pipeline stage (ingestion, adapter, validation, persistence)
- Add API endpoint tests covering success and failure paths
- Add validation and persistence unit tests
- Ensure all tests can be discovered and run from a single command

## Expected files/modules
- `tests/` — expand existing test directory
- `tests/test_pipeline.py` — end-to-end pipeline integration tests
- `tests/test_api.py` — API endpoint tests
- `tests/test_validation.py` — extends existing validation tests
- `tests/test_persistence.py` — persistence layer tests

## Acceptance criteria
- Automated tests exist for all major system components
- Full test suite runs from a single command (e.g. `pytest`)
- Test failures are clearly reported with context

## Short implementation notes
Use `pytest` as the test runner. Ensure `tests/` has a consistent structure mirroring `modules/`. Use fixtures for shared setup (e.g. sample records, mock persistence). CI tests should pass on a clean environment with no manual steps beyond dependency installation.
