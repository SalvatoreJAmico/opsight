## Purpose
Add structured logging to pipeline execution.

## Responsibilities
Logging should include:
- pipeline start
- stage start/end
- record counts processed
- validation failures
- persistence results

Suggested output location:
- logs/pipeline_run.log

## Expected files/modules
- run_pipeline.py
- logs/pipeline_run.log (runtime output)
- Optional logging setup helper under modules/config/

## Acceptance criteria
- Pipeline execution produces readable structured logs
- Logs include stage boundaries and record counts
- Logs capture validation and persistence outcomes

## Short implementation notes
Use Python logging with a consistent formatter (timestamp + level + stage + message). Ensure logs/ is auto-created when missing and emit clear INFO/ERROR events per stage boundary.
