## Purpose
Create the main execution entrypoint that runs the Opsight pipeline end-to-end.

## Responsibilities
- Load source data using the ingestion module
- Run adapter normalization to produce canonical records
- Execute validation using the existing validation stack
- Persist validated records using the persistence manager
- Produce a pipeline execution summary

## Expected files/modules
- run_pipeline.py
- Integrations with:
  - modules/ingestion/ingestion.py
  - modules/adapter/adapter.py
  - modules/validation/validator.py
  - modules/persistence/persistence_manager.py

## Acceptance criteria
- Running python run_pipeline.py executes the full pipeline
- Data flows through ingestion -> adapter -> validation -> persistence
- A summary report is returned or printed

## Short implementation notes
Build run_pipeline.py as a thin orchestrator that composes existing module APIs. Keep stage logic explicit and linear, and produce a structured summary object at the end that is also printed for CLI usage.
