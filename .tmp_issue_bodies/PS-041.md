## Purpose
Define and implement the ordered pipeline stage execution logic.

Pipeline stages must execute in this order:
1. ingestion
2. adapter
3. validation
4. persistence

## Responsibilities
- Pass records between stages
- Ensure validation occurs before persistence
- Prevent invalid records from being persisted

## Expected files/modules
- run_pipeline.py
- Any supporting updates needed in:
  - modules/validation/validator.py
  - modules/persistence/persistence_manager.py

## Acceptance criteria
- Pipeline stage order is enforced
- Stage outputs are passed correctly to the next stage
- Invalid records are never sent to persistence

## Short implementation notes
Use explicit stage outputs (for example: ingested_records, adapted_records, validation_result) and feed them forward in-order. Add guard checks so persistence only receives validated records.
