## Purpose
Generate a summary report after each pipeline execution.

Example summary structure:
{
  "records_ingested": 1000,
  "records_valid": 980,
  "records_invalid": 20,
  "records_persisted": 980
}

## Responsibilities
- collect metrics from each stage
- produce a final execution summary

## Expected files/modules
- run_pipeline.py
- Optional metric helpers in validation/persistence modules

## Acceptance criteria
- pipeline returns or prints a run summary
- metrics accurately reflect pipeline execution

## Short implementation notes
Collect metrics from authoritative stage outputs rather than inferred values. Build the final summary as a plain dict that can be both printed and serialized as JSON.
