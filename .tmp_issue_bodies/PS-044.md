## Purpose
Implement centralized error handling for pipeline execution.

## Responsibilities
- capture stage failures
- prevent partial pipeline corruption
- log errors clearly
- safely terminate or recover pipeline execution

## Expected files/modules
- run_pipeline.py
- Integration with:
  - modules/persistence/storage_error.py
  - modules/persistence/persistence_manager.py
  - modules/validation/validator.py

## Acceptance criteria
- pipeline errors are handled gracefully
- error messages are logged and reported clearly
- unsafe downstream execution is prevented after stage failure

## Short implementation notes
Wrap each stage in targeted exception handling and add a top-level failure boundary for final reporting. Include stage name and root cause in logs and return a failure summary payload for diagnostics.
